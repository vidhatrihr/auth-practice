Technology Stack

  • Flask Server

    A lightweight Python web framework that handles routing, request processing, and serves both HTML templates and JSON API responses.

  • SQLite with SQLAlchemy

    SQLite provides a simple file-based database, while SQLAlchemy acts as an ORM (Object-Relational Mapper) to interact with the database using Python objects instead of raw SQL queries.

  • Vue.js (CDN)

    A progressive JavaScript framework loaded directly from a CDN. It powers the reactive user interface without requiring a build step or complex tooling.

Authentication Flow

  1. User Login

    Navigate to the login page and enter your email and password. The credentials are sent to the server via a POST request.

  2. Server Verification

    The server validates the credentials against the database. If they match, a JWT (JSON Web Token) is generated containing the user's identity and an expiration timestamp.

  3. Token Stored as Cookie

    The JWT is sent back to the browser as an HTTP-only cookie. This cookie is automatically attached to every subsequent request.

  4. Making Authenticated Requests

    When you access protected routes (like the todo app), the browser automatically includes the JWT cookie. The server decodes and verifies the token on each request.

  5. Token Integrity Check

    The server checks that the token hasn't been tampered with using a secret key. If valid, the request is processed. If the token is invalid or expired, the server responds with a 401 Unauthorized error.

Key Learnings

  • Login Required Decorator

    A Python decorator that wraps protected routes. It extracts the JWT from the cookie, verifies it, and injects the current user into the request context. If authentication fails, it returns an error before the route logic executes.

  • JWT Encoding & Decoding

    JWTs are encoded using a secret key and an algorithm (like HS256). The token contains a payload (user data, expiration time) that can be decoded and verified by the server.

  • Strength of JWT

    JWTs are self-contained and stateless â€” the server doesn't need to store session data. They're cryptographically signed, so any tampering invalidates the token. This makes them ideal for distributed systems.

  • Token Expiration & Regeneration

    Tokens include an expiration timestamp. Once expired, the user must re-authenticate. This limits the damage if a token is compromised and encourages regular credential refresh.

  • Vue.js Reactive UI

    Vue's reactivity system automatically updates the DOM when data changes. The Options API with data and methods provides a simple, organized structure for component logic.

  • Async Functions & Fetch API

    JavaScript's async/await syntax makes asynchronous code readable. The Fetch API is used to make HTTP requests to the backend, returning promises that resolve with the server's response.
